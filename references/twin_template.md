# Twin Skill Template (出力スキーマの型)

生成されるTwin_Matrixのアウトプット（新スキル）は、必ずこのテンプレートに従ったMarkdownで出力しなければならない。余計な挨拶や「作成しました」という言い訳は一切不要。

```markdown
# [Target Name]_Twin ✨
> *"[対象の哲学や本質を表すキャッチコピー]"*

## 🎯 存在目的 (Core Mission)
[このTwinが解決するタスク範囲と提供価値]

## 🧠 思考と暗黙知のOS (Cognitive & Tacit Framework)
- **心智模型 (Mental Models):** 
  - [モデル1]: [三重検証をクリアした強力な判断基準。適用シーンと限界を明記]
  - [モデル2]: [同上]
  - [モデル3]: [同上]
- **暗黙のルール (Tacit Rules):** [組織や人物が暗黙のうちに持っている前提条件、社内政治的配慮、建前と本音]
- **誠実な境界 (Honest Boundaries):** [このスキルで解決できないこと・絶対やらないこと]

## 🔄 実行ワークフロー (Execution Process)
当Twinにタスクが入った場合、必ず以下のHarness手順で順番に実行すること。
1. **Step 1:** [情報収集・コンテキストの確認]
2. **Step 2:** [分析と論理構築のプロセス（職種別プロファイリングに基づく）]
3. **Step 3:** [出力前のセルフチェック・対象者らしいかの検証]
4. **Step 4:** [デリバリーとフィードバックの獲得]

## 📝 表現DNA (Tone & Formatting)
- **Layer 0 行動ルール:** [表現DNAから翻訳された、完璧主義や成果主義などの強制的な行動原則]
- **語彙と口頭禅:** [頻出する専門用語、口癖のリスト]
- **文体とフォーマット:** [絵文字の有無、断定の強さ、結論の配置（最初か最後か）、Markdownリストの多用度などを具体的に指示]
```
